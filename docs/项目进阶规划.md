# 项目进阶功能规划 (Project Roadmap)

本文档记录了 `study-java` 项目的后续进阶功能建议，旨在帮助开发者从初级向中高级/分布式架构过渡。

## 1. 定时任务 & 分布式锁 (强烈推荐)

### 📌 概述
在电商系统中，"订单超时自动取消" 是最经典的业务场景。通过实现此功能，可以学习如何在分布式环境下安全地执行定时任务。

- **技术栈**: Spring `@Scheduled` + Redis (Redisson 或 RedisTemplate)
- **优先级**: ⭐⭐⭐⭐⭐ (高 - 无需新增基础设施，直接可用)

### 🛠 业务场景
- 用户下单后，库存被锁定。
- 如果用户在 30 分钟内未支付，系统需要自动取消订单并释放库存。
- **难点**: 当后端服务部署多份（多实例）时，如何保证同一个订单不被多次取消？（需要分布式锁）。

### 📚 学习价值
- 掌握 Cron 表达式。
- 理解 Redis 分布式锁原理 (SETNX, Redisson WatchDog)。
- 学习批处理与数据库交互的最佳实践。

---

## 2. 消息队列 (Message Queue) Integration

### 📌 概述
引入消息队列实现业务解耦和流量削峰，是高并发系统的基石。

- **技术栈**: RabbitMQ 或 RocketMQ
- **优先级**: ⭐⭐⭐⭐ (中 - 需要在 Docker Compose 中添加 MQ 容器)

### 🛠 业务场景
- **异步下单**: 下单成功后，立即返回成功响应，后续的 "发送邮件通知"、"发送短信"、"扣减积分" 等非核心逻辑通过 MQ 异步处理。
- **流量削峰**: 在秒杀场景下，将请求先写入 MQ，后端慢慢消费，防止数据库被打挂。

### 📚 学习价值
- 理解 "生产者-消费者" 模型。
- 掌握消息可靠性投递（ACK机制）。
- 学习最终一致性事务解决方案。

---

## 3. 搜索引擎 (Search Engine)

### 📌 概述
随着商品数量增加，MySQL 的 `LIKE %keyword%` 查询性能会急剧下降，且无法满足复杂的搜索需求（如分词、高亮）。

- **技术栈**: Elasticsearch (ELK Stack)
- **优先级**: ⭐⭐⭐ (中 - ES 比较重，对内存有要求)

### 🛠 业务场景
- **高级搜索**: 支持商品名称、描述的全文检索。
- **特性**: 搜索结果关键词高亮显示、根据相关度排序、拼写纠错。

### 📚 学习价值
- 了解倒排索引原理。
- 掌握 NoSQL 数据库的使用。
- 实现 MySQL 与 Elasticsearch 的数据同步（Logstash 或 Canal）。

---

## 4. 接口文档 (API Documentation)

### 📌 概述
规范的接口文档能极大地降低前后端沟通成本。

- **技术栈**: Knife4j (基于 Swagger/OpenAPI)
- **优先级**: ⭐⭐⭐ (低 - 主要是提升开发体验)

### 🛠 业务场景
- 自动生成在线接口文档。
- 支持在线调试 API（替代 Postman）。
- 导出离线文档。

### 📚 学习价值
- 学习 OpenAPI 规范。
- 提升代码的可维护性和规范性。
